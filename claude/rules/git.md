# Git

- Never push to main
- Never include mentions of Claude Code in commits or MR descriptions (ex. co-authored by...)
- Use conventional commits
- Create atomic commits as much as possible
- Always create MRs as drafts
- Always push after creating a commit before working on the next change/commits
- If changes to be made are out of scope or are drastically different than what is worked on, ask the user whether a git worktree would be useful
- After each commit, read the MR description and update it
- **When implementing a plan, stop after each step and explain the changes. I will most likely want to create a commit and push but check with me first. Do NOT continue to the next step without my confirmation.**

## MR Description Preferences

- Use bullet points as much as possible
- Include visuals if necessary
- Check the current MR description for manual changes before overwriting
